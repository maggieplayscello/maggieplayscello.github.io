Tech Elevator Capstones

Capstone 1: Vending Machine CLI Program

View Code in GitHub

Completed in partnership with Andorra Fields.

  • Reads in an initial inventory CSV file.
  • CLI displays an initial menu of items, their price, and quantity.
  • Allows a user to deposit money and keeps a running total as they make purchases or additional deposits.
  • Updates the quantity remaining of an item after purchase; once it is sold out it will replace the name with the "SOLD OUT" message and will not allow you to purchase more.
  • User can cash out and exit; the system will inform user of the amount of refund and break down change into logical coin values for fewest coins.
  • Outputs a purchase log with timestamps.

Capstone 2: National Parks Campsite Registration

View Code in GitHub

Completed in partnership with Kevin Ventura.

  • Connects to a a SQL relational database using JDBC.
  • CLI displays an initial menu of three national park options.
  • Upon selection, formatted information about the park, drawn from the database, is displayed, along with a followup menu to allow a user to either view that park's campgrounds, search for a reservation, or exit to main menu.
  • View Campgrounds will display relevant info and offer an option to make a reservation.
  • Reservations can only be made during a park's open season and cannot overlap an existing reservation -- an appropriate message and redirect option will display.
  • If a reservation is available for a user's selected dates, up to five available campsites will be displayed, along with their details and the total cost of the full stay.
  • Upon entering a reservation name, a confirmation message will appear with a confirmation number.

Capstone 3: National Parks "Park Geek" Web Application

View Code in GitHub

Completed in partnership with S. Michael Robinson.

  • Connects to a a SQL relational database (3 tables) using JDBC.
  • Uses Spring MVC framework to create a functional web app.
  • JSPs (5, plus header) use HTML and EL. Also implements a site-wide header and CSS.
  • Home page displays a list of parks with images and short descriptions.
  • Clicking on a park will display additional information, including a 5 day weather forecast along with logical advice about how to dress and pack based on the predicted weather.
  • Weather can be displayed in either Fahrenheit or Celcius, and a user's preference will stay in effect as they navigate between park pages and displays in the header.
  • There is a survey page which takes basic user information, including their email address and favorite park.
  • A duplicate email submission results in a "Try again" page. An obviously invalid email will result in a "Please enter a valid email address" message.
  • Once a park has at least one vote, it will appear on the "Top Parks" page with its vote total.

Capstone 4: "Golf Tracker" App

View Code in GitHub

Completed entirely remotely in a team with Ryan Schudlich, Kevin Ventura, and Norm Rothwell.

My original wireframe design for the My Leagues page, alongside its finalized version.

  • Completed entirely remotely in less than two weeks by a four-person team in an Agile environment with a product owner, scrum master, and daily standups.
  • Team used Zoom calls, Ryver messaging, and an Asana board to communicate and coordinate.
  • Connects to a team-designed relational database (9 tables).
  • Uses Spring MVC framework to create a functional web app.
  • JSPs (22, plus header and footer) use HTML, EL, JS, local CSS, and Bootstrap. Also implements a site-wide header and footer and a site.CSS.
  • Designed wireframes, selected a color scheme and designed a logo. Used CSS and Bootstrap to accomplish design elements.
  • Unlogged-in users can use the course search, which includes a weather widget and a map of all courses matching search results.
  • Courses can be searched by name, city, and with full or partial words.
  • New users can register an account.
  • Existing users can also change their password.
  • Three tiers of privileges: golfer, league admin, and site admin.
  • All users can log personal scores and schedule tee times.
  • Tee times that are upcoming display in the dashboard with all relevant info. Scheduling a new tee time will display all courses in database, dates for three weeks out, and available times (crosschecked with existing reservations).
  • Tee time scheduler will display a warning when a user already has a tee time on their selected day.
  • Scores only require the course, par, and date. Par is an incrementer initially set at 72. Date is a bootstrap datepicker that will not allow future dates to be selected.
  • Once a user has at least 5 scores saved, a handicap will be auto-generated.
  • Users who are part of a league will have access to information about that league, including their own ranking and score, and to the roster of teams and list of team members.
  • League admins can create new leagues and add players to any existing league. They can also record match information.
  • Dashboard and My Leagues pages automatically update any newly entered data. My Leagues also allows a user to toggle in-page among leagues in which they are a member.
